Loci Interfaces

Any/all environment, arguments, and files which we use are listed here.

Database File

Purpose: record slow-moving (3-4 second latency) global state across app runs.

Location: %AppData%/Loci/db/db.db | $HOME/.local/share/Loci/db/db.db

Relevant Loci code: src/db/mod.rs Relevant 3rdparty code: None

Eapp Directory

Purpose: record slow-moving (3-4 second latency) global state across app runs.

Location: %AppData%/Loci/eapp | $HOME/.cache/Loci/eapp

Relevant Loci code: src/subprograms/mod.rs Relevant 3rdparty code: None

Environment Variables

RUST_BACKTRACE: - Part of the code generated by rustc when compiling indebug mode - set to 1 or full to get stack traces on panic!()s.

LOCI_DISABLED_SUBPROGRAMS: - List of subprogram names to disable; each subprogram essentially checks if NAME in LOCI_DISABLED_SUBPROGRAMS return

NO_CONSOLE_DETATCH: - if set windows systems will not detatch from CMD.exe when GetConsoleProcessList returns 1 (heuristic for "common GUI launches like double-clicking a .exe from explore.exe") - Useful for debugging, but not as useful as running loci.exe from an existing cmd.exe session.

LOCI_NO_GUI: - If set the main GUI is replaced with a loop { }, which makes testing and server demos easier.

LOCI_NO_ADMIN: - If set loci will not attempt to use admin privileges for anything. This will mostly affect radios which need admin privileges for communication.

LOCI_LICENSE_TXT: - If a license file does not exist this will be checked for a PGP-signed license in src/license.rs.
